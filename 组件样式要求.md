组件样式硬性要求：

本项目所有组件样式必须参考并复刻以下三个开源组件体系的设计语言，不允许自行发明一套新 UI 风格：

1. shadcn/ui
参考地址：
https://ui.shadcn.com/docs/components

用途：
- 作为 react-ui 基础组件的主要样式来源
- Button、Input、Textarea、Card、Badge、Dialog、Tabs、Empty、Table、Dropdown、Tooltip 等基础组件必须优先参考 shadcn/ui 的结构、圆角、间距、边框、hover、focus、disabled、dark mode、class 命名思路
- 使用 class-variance-authority / cn / Tailwind class 的组织方式
- 组件 API 保持简洁，接近 shadcn/ui 的使用体验
- 不要做花哨视觉，不要引入复杂动效

2. AI Elements
参考地址：
https://elements.ai-sdk.dev/components/attachments

用途：
- 作为 react-ai 中 AI 场景组件的主要样式来源之一
- 附件、消息、Prompt Input、Reasoning、Sources、Tool、Task、Code Block、Conversation 等 AI 组件优先参考 AI Elements 的组件组织方式
- 组件要保持 composable，可组合，而不是写成一个巨大黑盒
- 附件组件要支持 grid / inline / list 三种展示模式
- 工具调用、引用资料、文件附件、思考过程要使用克制的边框、浅灰背景、圆角卡片、紧凑信息层级
- 不直接调用模型 API，只做展示和交互

3. assistant-ui
参考地址：
https://www.assistant-ui.com/examples

用途：
- 作为完整 Chat / Assistant 页面体验的参考
- ChatPanel、MessageList、MessageItem、Composer、Thread、Assistant Sidebar、Floating Assistant、Copilot Layout 等交互和布局参考 assistant-ui examples
- playground-react 里的 AI 对话页面要参考 assistant-ui 的 ChatGPT Clone / Claude Clone / Modal / Form Filling Co-Pilot / Artifacts 示例
- 对话区要有清晰的用户消息、助手消息、输入区、工具调用状态、引用资料、附件、流式状态
- 整体体验像成熟 AI Assistant，而不是普通表单页面

视觉基调：
- 仍然以黑白灰为主
- shadcn/ui 作为基础视觉底座
- AI Elements 作为 AI 组件局部样式参考
- assistant-ui 作为完整助手布局和交互参考
- 保留极少量状态色：success / warning / error / info
- 状态色只用于反馈，不用于品牌装饰
- 禁止大面积彩色渐变
- 禁止高饱和装饰色
- 禁止卡通化、游戏化、花哨阴影
- 禁止每个组件自行定义颜色，必须使用 tokens 和 CSS Variables

样式实现要求：
- 所有颜色必须来自 @zero/tokens 或 CSS Variables
- Tailwind 颜色必须映射到 CSS Variables
- react-ui 样式优先对齐 shadcn/ui
- react-ai 样式优先对齐 AI Elements 和 assistant-ui
- playground-react 页面布局优先参考 assistant-ui examples
- 不要直接复制第三方库源码，除非许可证允许且保留必要说明
- 可以学习其组件结构、交互模式、样式组织方式
- 若使用 shadcn/ui CLI 添加组件，必须把组件整理进 @zero/react-ui
- 若参考 AI Elements 组件，必须改造成 @zero/react-ai 内部组件
- 若参考 assistant-ui 示例，必须改造成 playground-react 示例或 react-ai 组件

基础组件样式映射：

Button：
- 参考 shadcn/ui Button
- variant 至少包含：default / secondary / outline / ghost / destructive
- size 至少包含：sm / md / lg / icon

Input / Textarea：
- 参考 shadcn/ui Input / Textarea
- 黑白灰边框
- focus 使用 ring
- disabled 使用浅灰背景和禁用文字

Card：
- 参考 shadcn/ui Card
- 白底、细边框、轻圆角
- 支持 Header / Title / Description / Content / Footer

Dialog：
- 参考 shadcn/ui Dialog
- overlay、content、title、description、footer 结构清晰
- 不做复杂动画

Tabs：
- 参考 shadcn/ui Tabs
- 简洁分段切换
- 适合后台和 AI 工作台

Badge：
- 参考 shadcn/ui Badge
- 支持 default / secondary / outline / destructive

Empty：
- 参考 shadcn/ui Empty
- 用于空状态，不要花哨插画

AI 组件样式映射：

ChatPanel：
- 参考 assistant-ui ChatGPT Clone / Claude Clone 的整体体验
- 页面结构包含 Header、MessageList、Composer
- 留白克制，信息密度适中

MessageList：
- 参考 assistant-ui Thread 体验
- 支持滚动区域
- 消息之间留白清晰

MessageItem：
- 用户消息和助手消息有明显层级，但不要大面积彩色
- 用户消息可使用浅灰背景或右侧气泡
- 助手消息使用白底或透明底
- 支持 streaming / done / error

Composer：
- 参考 AI Elements Prompt Input 和 assistant-ui 输入区
- 支持多行输入
- 支持附件区域
- 支持发送按钮
- 支持 disabled / loading

Attachments：
- 参考 AI Elements Attachments
- 支持 grid / inline / list
- 支持 AttachmentPreview、AttachmentInfo、AttachmentRemove
- 图片、PDF、音频、视频、链接资料要有不同但克制的展示方式

ToolCallCard：
- 参考 AI Elements Tool
- 使用浅灰背景、边框、状态 Badge
- 支持 pending / running / success / error
- 可折叠展示参数和结果

CitationList：
- 参考 AI Elements Sources / Inline Citation
- 使用灰色边框、编号、来源标题
- 不使用复杂卡片装饰

ReasoningBlock：
- 参考 AI Elements Reasoning / Chain of Thought
- 默认可折叠
- 使用浅灰背景和细边框
- 不要把内容做得过于显眼

TaskTimeline：
- 参考 AI Elements Task / Plan
- 用于展示多步骤 AI 任务进度
- 使用黑白灰线条和状态点

playground-react 示例要求：
- 首页展示一个完整 AI Assistant 示例
- 视觉参考 assistant-ui 的 ChatGPT Clone / Claude Clone
- 包含 mock 消息
- 包含用户消息、助手消息、流式状态、工具调用、引用资料、附件、ReasoningBlock
- 不接真实模型 API
- 不写后端
- 重点展示组件体系和样式复用
